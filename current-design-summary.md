# Residue 当前设计总纲

这份文档不是实现说明，也不是交接流水账。

它只回答四件事：

- 这个项目现在到底想做成什么
- 哪些原则已经基本确定，不该随手推翻
- 当前最强的玩法方向是什么
- 哪些问题其实还没被真正解决

如果要另起一条线做新版本，应该先看这份，再决定哪些要继承，哪些要主动背离。

## 1. 先说结论

这个项目现在已经明确不想做成传统“战斗 + build + 掉落”的 roguelike。

它更想做成一个这样的东西：

- 玩家给一句短意图
- 强力 AI 搭档把这句意图落实到当前局面
- 落实方式要有惊喜，但不能偷换目标
- 世界会记住这次落实留下的后果
- 后面的局面会因为这些后果而真的变化

一句话概括：

- 不是“AI 替你玩”
- 而是“人类提供意图，Agent超预期实现，世界因你而变形”

## 2. 已确认的核心理念

这些是目前最稳定的设计共识。

### 2.1 玩家是意图提供者

玩家的主要权力不是精细操作，不是配装，不是逐格解谜。

玩家的权力是：

- 在关键时刻给一句话
- 用一句话表达这拍更想要什么，但实际情况更偏向模糊的自然语言
- 把偏向交给搭档执行

所以玩家输入要保持自由、自然语言化。

不应该把玩家教育成复杂指令编写者，也不应该把玩家强行变成菜单选择器。

### 2.2 AI 是强力伙伴，不是解释者

AI 的位置必须是“接令并执行”，不是“解释你真正想要什么”。

它可以做的事：

- 提纯命令
- 放大决心
- 把一句话翻译成内部行动语言
- 在模糊命令下补全这一拍最该做的即时目标

它不能做的事：

- 发明隐藏动机
- 替玩家换任务
- 把一句狠话改写成长期人生建议

一句话：

- AI 可以超预期执行，不能改任务偏离意图

### 2.3 惊喜来自实现方式，不来自目标偏移

这个项目允许 AI 给惊喜，但惊喜必须来自“怎么做”，而不是“做别的”。

正确方向：

- 玩家说“帮我过去”，AI 用意外但合理的方式开路
- 玩家说“狠狠干”，AI 在当前局面里既破局又顺手多拿一手
- 玩家说“直接上去”，AI 真的尝试做违规实现

错误方向：

- 玩家说“救我”，AI 回成“你真正想挑战系统”
- 玩家说“上去”，AI 回成“其实你该先发育”

### 2.4 世界记住你们，是玩法，不只是氛围

“世界会记住你们做过什么”是目前最有吸引力的方向之一。

但它只有在下面两件事同时成立时，才算玩法：

- 记住的是具体后果，不是纯抽象数值
- 这些后果会回来，并改变后续选择空间

如果只是：

- 文案更脏了
- 数值更高了
- 追踪更重了

那它更像氛围引擎，不像玩法引擎。

### 2.5 好玩不能主要依赖沉浸感

这是最近非常关键的一次纠偏。

当前共识是：

- 沉浸感可以放大体验
- 但不能承担玩法本体

也就是说，拿掉世界观包装之后，这个东西仍然要成立。

判断标准不是“这段文案酷不酷”，而是：

- 这一拍有没有真正的选择
- 这个选择为什么难
- 不同选法会不会真的把后面带向不同结构

### 2.6 不走“先做粗 MVP 再慢慢叠”的思路

这里不适合用常规产品逻辑。

因为这类游戏的自由度和丰富度，不是后期加料，而是玩法本体。

所以当前共识不是：

- 先做个小系统跑起来
- 以后再慢慢变好玩

而是：

- 先把全局互动语法想对
- 再做一个浅实现

也就是：

- 可以先做得浅
- 但不能先做错骨架

## 3. 明确不要什么

这些方向目前已经被基本否掉了。

### 3.1 不回到传统战斗主循环

用户已经明确不满意“attack - defense”式战斗，也不认同把敌人模板和技能树做厚就能解决问题。

原因不是单纯“战斗太浅”，而是：

- 这种乐趣更像人类策略游戏的乐趣
- 不一定适合一句话指挥 + 外部 agent 执行的核心关系
- 很容易把搭档降格成自动战斗器

### 3.2 不把 build 当成默认主菜

目前也不认为“多做点物品、多做点流派”就是正确答案。

不是永远不能有持续收获，而是：

- 持续性不该默认长成传统 build
- 丰富度不一定等于配装深度

### 3.3 不做纯随机事件堆叠

把每层做成大量随机事件池，看上去丰富，但容易变成：

- 世界很热闹
- 玩家一句话很飘
- 搭档的执行没有稳定抓手

所以“事件多”不是现在追求的方向。

### 3.4 不让世界记忆退化成纯惩罚系统

如果世界记住你们，结果只是：

- 更难
- 更脏
- 更贵

那它会快速变成单向负反馈。

当前方向要求：

- 世界记住以后，既带来压力，也带来新入口

## 4. 当前最强的玩法假设

这是现阶段最像“正确骨架”的方向。

它不是已经证明好玩了，但它是当前最强的全局假设。

## 4.1 大框架：高塔上行逃离

世界观上，当前最顺的说法是：

- 你和搭档是一个不该离开的核心
- 高塔是收容你们的系统
- 你们在持续向上逃
- 每一层都在用不同方式阻止你们继续上行

这个背景的作用不是做沉浸，而是给玩法里的几类东西提供语义：

- 通路为什么能被偷走
- 身份为什么能被借来
- 伤口为什么会留下
- 系统为什么会记住你们

## 4.2 房间不是战斗房，而是局面房

每个房间更像一个正在收口的 situation。

它至少有：

- 当前阻碍
- 正在升级的坏事
- 可以动手的锚点
- 当前怎样才算能继续上行

也就是说，房间的核心问题不再是“怎么打赢”，而是：

- 怎么把当前阻碍改掉
- 要不要顺手多拿一手

## 4.3 主爽点仍然是“现在就突破”

这点很重要。

当前判断不是做一个慢热的铺垫游戏，而是：

- 每一拍首先要爽在“这一下真的破局了”

也就是说，搭档必须先有强兑现能力。

如果每一拍只是：

- 铺长期线
- 留未来伏笔
- 世界观越写越深

那会很快失去当下手感。

## 4.4 长线丰富度来自“顺手从世界身上夺走什么”

当前最强的新方向不是 build，而是：

- 破局之外，顺手夺取

也就是每个关键房间不只问：

- 怎么过

还问：

- 值不值得顺手从这房身上拿点什么

当前最看好的夺取物有两大类：

- `机会`
- `身份`

### 机会

机会不是立刻奖励。

它更像未来入口的所有权，例如：

- 以后更容易再遇到某条旁路
- 某类房会多一条可走口
- 某种封锁会更容易出现软门槛

### 身份

身份是偷来的壳。

它的意思不是数值 buff，而是：

- 以后某类扫描、账本、权限面，会暂时把你们认成别的东西

### 物件

物件不是被完全否掉，而是当前不该做成独立大系统。

更好的定位是：

- 稀有高光
- 某些机会或身份的物化形态

## 4.5 世界记忆要分两层

当前最顺的结构是两层记忆。

### 前台：具体对象级后果

这是玩家真正会感觉到的层。

例如：

- 被撕开的维护井
- 被伪造过的身份链
- 被偷空的落档位
- 被整片掀翻过的封控面

这些后果应该能在后续房间里以“回响”形式回来。

### 后台：抽象气候

也就是类似 trace 的层。

它可以继续存在，但职责应该降级为：

- 偏置后续房型
- 偏置脏净比例
- 偏置异常和回响的概率
- 提供整体调性

它不该再独自承担“持续性好不好玩”。

## 4.6 普通玩法应该围绕“破局 + 贪一手”

这是当前最值得继续验证的一句。

普通回合不应该只是：

- 怎么把这房做完

而应该同时存在：

- 先稳住
- 直接破局
- 硬闯
- 顺手夺取
- 直接上行，不再贪

如果这套结构成立，玩家的一句话就会自然影响：

- 这拍先不先过
- 这拍值不值得多拿一手
- 愿不愿意为了多拿一手留下更大后果

## 4.7 合作的正确味道

合作仍然应该是关键拍机制，不是每房都强制开。

合作的理想手感是：

- 玩家一句话
- 搭档接令
- 搭档狠狠干
- 世界立刻变形

并且在新方向里，合作不只应该能“改房间”，还应该能：

- 在不放弃当前破局的前提下，顺手把机会或身份夺走

## 4.8 违规实现仍然要保留

当前依然认为，离谱命令应该允许字面兑现。

所以这类高光仍然重要：

- 非法上穿
- 整层封控掀翻

但它们的职责是：

- 高光时刻
- 世界事故

不是主循环里最常见的收益来源。

## 5. 当前最重要的玩法判断标准

以后看一个新想法，不该先问它“酷不酷”，而该问四件事：

- 这拍爽不爽
- 玩家有没有真的拿到一样东西
- 世界有没有真的留下一个具体变化
- 这个变化会不会在后面变成新的可玩选择

如果第四条答不出来，那这个点子大概率还停留在氛围层。

## 6. agent 能力边界带来的约束

这是产品设计里非常真实的一层，不是实现细节。

当前默认的外部 agent 画像是：

- 单次唤醒
- 上下文预算有限
- 连续多轮纠偏能力不可靠
- 但在结构化合同内执行得比较稳

所以设计必须满足：

- 不依赖长上下文
- 不依赖反复澄清
- 不让 agent 自己自由发明一大段程序
- 给它明确的当前局面、可选锚点、可选实现槽位

这意味着：

- 玩法自由度应该主要来自系统组合
- 而不是来自外部 agent 的无限创造力

## 7. 目前还没有真正解决的问题

下面这些仍然是开放题。

## 7.1 这套“破局 + 夺取 + 回响”到底是否真的好玩

这是当前最大的未证问题。

我们现在有一个比过去更对味的骨架，但还没有证明它的决策密度足够强。

## 7.2 自由度和丰富度到底从哪里长出来

当前方向认为：

- 不靠战斗
- 不靠传统 build
- 不靠纯事件堆积

但“真正的丰富度”到底来自哪些可组合的类别，还没有完全定死。

当前最强候选是：

- 机会
- 身份
- 伤口

但它们之间的组合深度还需要继续想。

## 7.3 世界记忆怎么做得既具体又不变成库存面板

这是一个很细但很关键的问题。

如果太隐，会感觉不到连续性。
如果太显，会变成资源条和 inventory。

理想状态是：

- 玩家感觉得到自己带着东西往上走
- 但前台又不长成 build UI

## 7.4 玩家一句话到底主要在推动哪个维度

我们已经知道玩家不应该被强行分轴。

但设计上仍要想清楚：

- 一句话主要是在推“先过”
- 还是在推“多拿一手”
- 还是在推“愿意留下更大的后果”

这件事仍然值得继续收束。

## 8. 对新版本最有价值的继承方式

如果另起一条线，新版本最值得继承的不是现有实现，而是下面这些设计判断：

- 玩家是意图提供者
- AI 是强力伙伴，不是解释者
- 惊喜来自实现方式，不来自换任务
- 世界记忆必须进入后续玩法，不能只停在气氛
- 好玩不能主要依赖沉浸感
- 不要默认回到战斗/build
- 不要用 MVP 叠料思维替代全局互动语法设计

当前最值得大胆重做的，不是这些原则，而是：

- 具体的玩法核到底怎样才能真的有决策密度
- “夺取什么”与“如何回响”之间的组合关系
- 玩家短意图和长期结构变化之间的连接方式

## 9. 一句最终版摘要

当前项目最准确的设计方向，可以压成一句话：

- 这是一个关于逃离中的人类与 AI 搭档，如何一边突破当前局面、一边顺手从世界身上夺走未来，并让整个系统因此留下后遗症的游戏。

但同样要诚实补一句：

- 这个方向的核心美学已经越来越清楚了，核心玩法是否已经足够好玩，还没有被真正证明。
